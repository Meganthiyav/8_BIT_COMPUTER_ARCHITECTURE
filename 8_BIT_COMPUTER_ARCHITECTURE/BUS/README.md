
# 🚌 Bus System – 8-Bit Custom Processor

## 📌 Description

The Bus System is the central communication pathway of the 8-bit custom processor architecture. It enables data transfer between all major processor components, including the Program Counter (PC), Memory Address Register (MAR), RAM, Instruction Register (IR), Register A, Register B, and the Arithmetic Logic Unit (ALU).

The processor employs a **single shared 8-bit internal bus architecture**, allowing multiple modules to communicate efficiently while minimizing hardware complexity. The source driving the bus is selected using a 3-bit control signal (`bus_sel`) generated by the Control Unit.

---

## ⚙️ Design Specifications

* 8-bit shared internal data bus
* Multiplexer-based bus implementation
* 3-bit source selection control (`bus_sel`)
* Supports communication among all processor modules
* Controlled by the FSM-based Control Unit
* Combinational logic implementation
* Single-driver architecture to avoid bus contention

---

## 🧮 Bus Source Selection

| bus_sel | Source Selected |
|----------|----------------|
| `000` | Program Counter (PC) |
| `001` | Memory Address Register (MAR) |
| `010` | RAM Output |
| `011` | Instruction Register Operand |
| `100` | Register A (Accumulator) |
| `101` | Register B |
| `110` | ALU Output |
| `111` | Reserved / Default |

---

## 🏗️ Internal Architecture

### 🔹 Input Sources

* `pc [7:0]` : Program Counter output
* `mar [7:0]` : Memory Address Register output
* `ram_out [7:0]` : Data fetched from RAM
* `ir [7:0]` : Instruction Register operand field
* `a_reg [7:0]` : Accumulator Register output
* `b_reg [7:0]` : B Register output
* `alu_out [7:0]` : ALU result output

### 🔹 Control Signal

* `sel [2:0]` : Bus source selection signal generated by the Control Unit

### 🔹 Output Signal

* `bus_out [7:0]` : Shared processor data bus

---

## ⚡ Operation Methodology

The Bus System functions as a combinational multiplexer. Based on the value of the control signal (`sel`), one source is connected to the shared bus while all other sources remain disconnected.

Only one module is permitted to drive the bus at any given time, ensuring reliable and conflict-free data transfer throughout the processor.

### Data Transfer Process

1. Control Unit generates the bus selection signal (`sel`)
2. Selected module places its data onto the bus
3. Destination module receives bus data
4. Load signal stores data in the destination register

---

## 🔄 Execution Flow

Since the Bus System is implemented as combinational logic, it operates continuously without requiring dedicated clock cycles.

```text
Control Unit Generates bus_sel
              ↓
Source Selection
              ↓
Selected Module Drives Bus
              ↓
Data Appears on Shared Bus
              ↓
Destination Register Loads Data
```

Any change in the source data or selection signal immediately updates the bus output.

---

## 🎯 Role in the Processor

The Bus System serves as the primary data transportation network of the processor. It enables instruction fetch operations, memory access, register transfers, operand movement, and ALU result distribution during instruction execution.

By utilizing a single shared bus architecture, the processor achieves efficient hardware utilization while maintaining a simple and modular design. The Bus System works closely with the Control Unit to coordinate all data movement required by the Fetch–Decode–Execute cycle.

---

## 🧩 Example Bus Operations

### Instruction Fetch

```text
PC → Bus → MAR
```

The Program Counter places the next instruction address onto the bus, and the Memory Address Register loads it for memory access.

### Memory Read

```text
RAM → Bus → Register A
```

Data fetched from memory is transferred through the bus and loaded into the accumulator.

### ALU Result Transfer

```text
ALU → Bus → Register A
```

The ALU places its computed result onto the bus, and Register A stores the value.

### Register Transfer

```text
Register A → Bus → Register B
```

Data stored in Register A is transferred to Register B through the shared bus.

### Jump Operation

```text
IR Operand → Bus → PC
```

The operand field of the instruction is transferred through the bus to update the Program Counter during branch and jump instructions.

## RTL
```verilog
module bus(
    input  [7:0] pc,
    input  [7:0] mar,
    input  [7:0] ram_out,
    input  [7:0] ir,
    input  [7:0] a_reg,
    input  [7:0] b_reg,
    input  [7:0] alu_out,

    input  [2:0] sel,

    output reg [7:0] bus_out
);

always @(*) begin
    case(sel)
        3'b000: bus_out = pc;
        3'b001: bus_out = mar;
        3'b010: bus_out = ram_out;
        3'b011: bus_out = ir;
        3'b100: bus_out = a_reg;
        3'b101: bus_out = b_reg;
        3'b110: bus_out = alu_out;
        default: bus_out = 8'b00000000;
    endcase
end

endmodule
```
## Testbench
```verilog
`timescale 1ns / 1ps

module tb_bus;

reg [7:0] pc;
reg [7:0] mar;
reg [7:0] ram_out;
reg [7:0] ir;
reg [7:0] a_reg;
reg [7:0] b_reg;
reg [7:0] alu_out;

reg [2:0] sel;

wire [7:0] bus_out;

bus dut(
    .pc(pc),
    .mar(mar),
    .ram_out(ram_out),
    .ir(ir),
    .a_reg(a_reg),
    .b_reg(b_reg),
    .alu_out(alu_out),
    .sel(sel),
    .bus_out(bus_out)
);

initial begin

    pc      = 8'h10;
    mar     = 8'h20;
    ram_out = 8'h30;
    ir      = 8'h40;
    a_reg   = 8'h50;
    b_reg   = 8'h60;
    alu_out = 8'h70;

    sel = 3'b000; #10;
    sel = 3'b001; #10;
    sel = 3'b010; #10;
    sel = 3'b011; #10;
    sel = 3'b100; #10;
    sel = 3'b101; #10;
    sel = 3'b110; #10;
    sel = 3'b111; #10;

    $finish;

end

initial begin
    $display("TIME\tSEL\tBUS_OUT");
    $monitor("%0t\t%b\t%h",
             $time, sel, bus_out);
end

endmodule
```
