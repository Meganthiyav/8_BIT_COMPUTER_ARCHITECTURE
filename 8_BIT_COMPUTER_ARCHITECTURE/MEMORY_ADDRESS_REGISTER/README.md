# 📦 Memory Address Register (MAR) – 8-Bit Custom Processor

## 📌 Description

The **Memory Address Register (MAR)** is a dedicated address-holding register responsible for supplying memory addresses to the RAM during instruction fetch and data access operations. It serves as the interface between the processor datapath and the memory subsystem by temporarily storing the address of the memory location to be accessed.

In the 8-bit custom processor architecture, the MAR is implemented as a **4-bit synchronous register**, allowing access to **16 memory locations (0–15)**. The register receives address information from the system bus under the control of the Control Unit and continuously drives the memory address lines until a new address is loaded.

The MAR is an essential component of the fetch-execute cycle, ensuring accurate memory addressing and reliable communication between the processor and memory.

---

## ⚙️ Design Specifications

* 4-bit address register
* Synchronous operation using system clock
* Load-enable controlled address storage
* Reset support for initialization
* Direct connection to memory address bus
* Supports instruction fetch and operand access cycles
* Compatible with 16×8 memory organization
* Controlled by the Control Unit through dedicated load signals

---

## 🏗️ Internal Architecture

### 🔹 Input Signals

* `clk` : System clock
* `rst` : Reset signal
* `mar_load` : Load enable signal
* `address_in [3:0]` : Address input from the processor bus

### 🔹 Internal Storage

* `mar_reg [3:0]` : Internal register used to store the selected memory address

### 🔹 Output Signals

* `address_out [3:0]` : Address provided to the memory address lines

---

## ⚡ Operation Methodology

The Memory Address Register captures and stores an address value from the processor bus whenever the Control Unit asserts the load signal. The stored address is then presented to the memory subsystem for instruction retrieval or data access operations.

### Address Loading

When the load control signal is asserted, the address available on the processor bus is loaded into the MAR on the active clock edge.

### Address Retention

When the load signal is deasserted, the register maintains its current address value, ensuring a stable memory address throughout the memory access operation.

### Reset Operation

Upon activation of the reset signal, the register is cleared to a known state, initializing the memory addressing mechanism during processor startup.

---

## 🔄 Execution Flow

```text
Address Source
(PC / IR Address Field)
          ↓
       Processor Bus
          ↓
         MAR
          ↓
   Memory Address Bus
          ↓
          RAM
```

The address is first generated by the Program Counter or the address field of an instruction. The Control Unit loads this address into the MAR, which then drives the memory address bus and selects the corresponding memory location.

---

## ⏱️ MAR Operation During Processor Execution

### Instruction Fetch Phase

```text
PC → Bus → MAR
MAR → Memory Address Lines
RAM → Instruction Data
```

The Program Counter provides the address of the next instruction, which is loaded into the MAR. The memory location identified by the MAR is then accessed to retrieve the instruction.

### Operand Access Phase

```text
Instruction Address Field
            ↓
           MAR
            ↓
           RAM
            ↓
       Operand Data
```

For memory-reference instructions, the address field contained within the instruction is loaded into the MAR. The selected memory location is subsequently accessed to obtain the required operand.

---

## 🎯 Role in the Processor

The Memory Address Register forms the processor's addressing interface and plays a fundamental role in all memory transactions. By providing a stable and controlled address to the memory subsystem, it enables efficient instruction fetching, operand retrieval, and overall processor operation.

The MAR works in coordination with the Program Counter, Instruction Register, RAM, system bus, and Control Unit to ensure accurate memory addressing throughout the execution cycle of the processor.

## RTL
```verilog
module ram(
    input clk,
    input write_en,
    input [3:0] address,
    input [7:0] data_in,
    output reg [7:0] data_out
);

reg [7:0] memory [0:15];

always @(posedge clk)
begin
    if(write_en)
        memory[address] <= data_in;

    data_out <= memory[address];
end

endmodule
```

```verilog
module ram_tb;

reg clk;
reg write_en;
reg [3:0] address;
reg [7:0] data_in;
wire [7:0] data_out;

ram uut(
    .clk(clk),
    .write_en(write_en),
    .address(address),
    .data_in(data_in),
    .data_out(data_out)
);

always #5 clk = ~clk;

initial
begin
    clk = 0;
    write_en = 0;
    address = 0;
    data_in = 0;
    #10;
    write_en = 1;
    address = 4'b0101;
    data_in = 8'b00001000;
    #10;
    write_en = 0;
    #10;
    address = 4'b0101;
    #10;
    write_en = 1;
    address = 4'b0110;
    data_in = 8'b00000011;
    #10;
    write_en = 0;
    #10;
    address = 4'b0110;
    #20;
    $finish;
end
initial
begin
    $monitor("Time=%0t write_en=%b address=%d data_in=%d data_out=%d",
              $time, write_en, address, data_in, data_out);
end

endmodule
```
