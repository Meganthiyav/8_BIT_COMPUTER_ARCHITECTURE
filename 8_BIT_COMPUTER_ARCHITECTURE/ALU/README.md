# 🧮 Arithmetic Logic Unit (ALU) – 8-Bit Custom Processor

## 📌 Description

The Arithmetic Logic Unit (ALU) is the computational core of the 8-bit custom processor architecture. It performs arithmetic and logical operations on two 8-bit operands based on a 3-bit control signal (`command_in`) generated by the Control Unit.

The ALU is implemented as a purely combinational module, ensuring that output results are updated immediately whenever the input operands or operation code change. In addition to generating the operation result, the ALU produces a carry flag used for status monitoring and arithmetic overflow detection.

---

## ⚙️ Design Specifications

* 8-bit input operands (`a_in`, `b_in`)
* 8-bit result output (`result`)
* 3-bit operation selection input (`command_in`)
* Carry/Borrow flag generation (`carry_out`)
* Combinational logic implementation using `always @(*)`
* 9-bit internal temporary register for overflow handling
* Compatible with the processor datapath and Control Unit

---

## 🧮 Supported Operations

| Command | Operation | Function |
| ------- | --------- | -------- |
| `000`   | ADD       | A + B    |
| `001`   | SUB       | A - B    |
| `010`   | INC       | A + 1    |
| `011`   | DEC       | A - 1    |
| `100`   | AND       | A & B    |
| `101`   | OR        | A | B    |
| `110`   | XOR       | A ^ B    |
| `111`   | NOT       | ~A       |

---

## 🏗️ Internal Architecture

### 🔹 Input Signals

* `a_in [7:0]` : First operand
* `b_in [7:0]` : Second operand
* `command_in [2:0]` : Operation select signal

### 🔹 Internal Storage

* `temp [8:0]` : Temporary register used to capture carry and overflow information during arithmetic operations

### 🔹 Output Signals

* `result [7:0]` : Computed operation result
* `carry_out` : Carry or borrow indication

---

## ⚡ Operation Methodology

For arithmetic operations, the ALU utilizes a 9-bit temporary register (`temp`) to preserve overflow information beyond the 8-bit result range.

### Arithmetic Operations

* Addition and Increment generate a carry when the result exceeds 8 bits.
* Subtraction generates a borrow indication when the subtrahend is larger than the minuend.
* Decrement operation performs subtraction by one.

### Logical Operations

* AND, OR, XOR, and NOT operations directly manipulate individual bits without affecting the carry flag.

### Carry Flag Behavior

| Operation        | Carry Output                |
| ---------------- | --------------------------- |
| ADD              | Overflow carry bit          |
| INC              | Overflow carry bit          |
| SUB              | Borrow indicator            |
| DEC              | Borrow/underflow indication |
| Logic Operations | 0                           |

---

## 🔄 Execution Flow

Since the ALU is a combinational circuit, it does not operate using clock cycles or processor T-states.

```text
Operand Input
      ↓
Operation Selection (command_in)
      ↓
ALU Computation
      ↓
Result Generation
      ↓
Carry Flag Update
```

Any change in the input operands or control signal immediately triggers recomputation of the output.

---

## 🎯 Role in the Processor

The ALU serves as the primary execution unit of the processor, enabling arithmetic calculations, logical decision-making, data manipulation, and status flag generation required during instruction execution. It works closely with the A Register, B Register, Flag Register, and Control Unit to execute processor instructions efficiently.

## RTL

```verilog
module alu (
    input  [7:0]  a_in,
    input  [7:0]  b_in,
    input  [3:0]  command_in,
    input         oe,
    output [15:0] d_out
);

    parameter ADD  = 4'b0000,
              INC  = 4'b0001,
              SUB  = 4'b0010,
              DEC  = 4'b0011,
              MUL  = 4'b0100,
              DIV  = 4'b0101,
              SHL  = 4'b0110,
              SHR  = 4'b0111,
              AND  = 4'b1000,
              OR   = 4'b1001,
              INV  = 4'b1010,
              NAND = 4'b1011,
              NOR  = 4'b1100,
              XOR  = 4'b1101,
              XNOR = 4'b1110,
              BUF  = 4'b1111;

    reg [15:0] out;

    always @(*) begin
        case (command_in)
            ADD  : out = a_in + b_in;
            INC  : out = a_in + 1'b1;
            SUB  : out = a_in - b_in;
            DEC  : out = a_in - 1'b1;
            MUL  : out = a_in * b_in;
            DIV  : out = (b_in != 0) ? (a_in / b_in) : 16'd0;
            SHL  : out = a_in << 1;
            SHR  : out = a_in >> 1;
            AND  : out = a_in & b_in;
            OR   : out = a_in | b_in;
            INV  : out = ~a_in;
            NAND : out = ~(a_in & b_in);
            NOR  : out = ~(a_in | b_in);
            XOR  : out = a_in ^ b_in;
            XNOR : out = a_in ~^ b_in;
            BUF  : out = a_in;
            default : out = 16'd0;
        endcase
    end

    assign d_out = (oe) ? out : 16'hZZZZ;

endmodule
```
## TESTBENCH
```verilog
module alu_tb();
    
   //Testbench global variables
   reg [7:0]a,b;
   reg [3:0]command;
   reg enable;
   wire [15:0]out;
	
   //Variables for iteration of the loops 
   integer m,n,o;

   //Parameter constants used for displaying the strings during operation
   parameter ADD  = 4'b0000, // Add two 8 bit numbers a and b.
  	     INC  = 4'b0001, // Increment a by 1. 
	     SUB  = 4'b0010, // Subtracts b from a.
	     DEC  = 4'b0011, // Decrement a by 1.
	     MUL  = 4'b0100, // Multiply 4 bit numbers a and b.
	     DIV  = 4'b0101, // Divide a by b.
	     SHL  = 4'b0110, // Shift a to left side by 1 bit.
	     SHR  = 4'b0111, // Shift a to right by 1 bit.
	     AND  = 4'b1000, // Logical AND operation
	     OR   = 4'b1001, // Logical OR operation
	     INV  = 4'b1010, // Logical Negation
	     NAND = 4'b1011, // Bitwise NAND
	     NOR  = 4'b1100, // Bitwise NOR
	     XOR  = 4'b1101, // Bitwise XOR
	     XNOR = 4'b1110, // Bitwise XNOR
	     BUF  = 4'b1111; // BUF
					
   //Internal register for storing the string values
   reg [4*8:0]string_cmd;

   //Step1 : Instantiate the design ALU
   alu DUT (a,
            b,
            command,
            enable,
            out);

   //Step2 : Write a task named "initialize" to initialize the inputs of DUT
   task initialize;
      begin
         {a,b,command,enable} = 0;
      end
   endtask


   //Tasks used for generating stimulus
   task en_oe(input i);
      begin
         enable=i;
      end
   endtask

   task inputs(input [7:0]j,k);
      begin
	 a=j;
	 b=k;
      end
   endtask

   task cmd (input [3:0]l);
      begin
         command=l;
      end
   endtask

   task delay();
      begin
	 #10;
      end
   endtask
		
   //Process to hold the string values as per the commands.
   always@(command)
      begin
         case (command)
	    ADD    :  string_cmd = "ADD";
	    INC    :  string_cmd = "INC";
	    SUB    :  string_cmd = "SUB";
	    DEC    :  string_cmd = "DEC";
	    MUL    :  string_cmd = "MUL";
	    DIV    :  string_cmd = "DIV";
	    SHL    :  string_cmd = "SHL";
	    SHR    :  string_cmd = "SHR";
	    INV    :  string_cmd = "INV";
	    AND    :  string_cmd = "AND";
	    OR     :  string_cmd = "OR";
	    NAND   :  string_cmd = "NAND";
	    NOR    :  string_cmd = "NOR";
	    XOR    :  string_cmd = "XOR";
	    XNOR   :  string_cmd = "XNOR";
	    BUF    :  string_cmd = "BUF";
	 endcase
      end
			
   //Process used for generating stimulus by calling tasks & passing values
   initial
      begin
	 initialize;
	 en_oe(1'b1);
	 for(m=0;m<16;m=m+1)
	    begin
	       for(n=0;n<16;n=n+1)
	          begin
	             inputs(m,n);
		     for(o=0;o<16;o=o+1)
			begin
			   command=o;
			   delay;
			end 
	          end
            end  
         en_oe(0);
         inputs(8'd20,8'd10);
         cmd(ADD);
         delay;
         en_oe(1);
         inputs(8'd25,8'd17);
         cmd(ADD);
         delay;  
         $finish;
      end
			
   //Process to monitor the changes in the variables	
   initial 
      $monitor("Input oe=%b, a=%b, b=%b, command=%s, Output out=%b",enable,a,b,string_cmd,out);
								
   
endmodule
```
