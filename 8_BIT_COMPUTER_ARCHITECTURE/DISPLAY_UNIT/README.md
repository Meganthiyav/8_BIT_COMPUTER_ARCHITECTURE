# 🖥️ Display Unit

## 📌 Overview

The Display Unit serves as the output interface of the 8-bit computer architecture. It receives processed data from the Output Register and presents the final result of instruction execution for observation and verification.

The module acts as the endpoint of the datapath, allowing users to monitor the current output generated by the processor.

---

## ⚙️ Specifications

* 8-bit output data support
* Receives data from the Output Register
* Real-time display of processed results
* Synchronous operation with system execution
* FPGA-compatible output interface
* Simple and lightweight hardware implementation

---

## 🔌 Inputs / Outputs

| Port        | Direction | Width | Description                             |
| ----------- | --------- | ----- | --------------------------------------- |
| data_in     | Input     | 8     | Data received from the Output Register  |
| display_out | Output    | 8     | Data presented to the display interface |

---

## 🔄 Operation

* The Display Unit continuously monitors the value stored in the Output Register.
* Whenever the Output Register is updated, the new value is reflected at the display output.
* The displayed value represents the final result of the executed instruction or computation.

---

## 🏛️ Role in Architecture

The Display Unit provides a visual representation of processor outputs and serves as the final stage of the data path. It enables users to verify arithmetic, logical, and data-transfer operations performed by the computer.

---

## 🧩 Architectural Integration

```text
A Register ─┐
            │
B Register ─┼──► ALU ───► Output Register ───► Display Unit
            │
 Memory  ───┘
```

---

## 🔗 Connected Modules

* Output Register
* Bus System
* Control Unit (indirectly through output control signals)

## RTL 
```verilog
`timescale 1ns / 1ps

// =====================================================
// TOP-LEVEL MODULE: BINARY-TO-DECIMAL CONTROLLER
// =====================================================
module computer_display_controller #(
    parameter SIMULATION = 0
)(
    input clk,
    input reset,
    input [7:0] computer_out, // 8-bit binary register input (0 to 255)
    
    output [7:0] seg,         // Active-High segments [DP, G, F, E, D, C, B, A]
    output [3:0] an           // Active-Low digit anodes [Digit3, Digit2, Digit1, Digit0]
);

    // BCD Digit storage wires
    reg [3:0] hundreds;
    reg [3:0] tens;
    reg [3:0] ones;
    wire [3:0] thousands = 4'd0; // Leftmost display stays 0

    // Binary to BCD conversion logic (Double Dabble)
    integer i;
    always @(*) begin
        hundreds = 4'd0;
        tens     = 4'd0;
        ones     = 4'd0;
        
        for (i = 7; i >= 0; i = i - 1) begin
            if (hundreds >= 5) hundreds = hundreds + 3;
            if (tens     >= 5) tens     = tens + 3;
            if (ones     >= 5) ones     = ones + 3;
            
            hundreds = (hundreds << 1) | tens;
            tens     = (tens     << 1) | ones;
            ones     = (ones     << 1) | computer_out[i];
        end
    end

    // Instantiation linking to the hardware segment driver below
    seven_seg_controller #(
        .SIMULATION(SIMULATION)
    ) display_driver (
        .clk(clk),
        .reset(reset),
        .digit0(ones),       // Right digit
        .digit1(tens),       // Middle digit
        .digit2(hundreds),   // Left digit
        .digit3(thousands),  // Blank leading digit
        .seg(seg),
        .an(an)
    );

endmodule


// =====================================================
// SUB-MODULE: HARDWARE 7-SEGMENT DRIVER
// =====================================================
module seven_seg_controller #(
    parameter SIMULATION = 0
)(
    input clk,
    input reset,
    input [3:0] digit0,
    input [3:0] digit1,
    input [3:0] digit2,
    input [3:0] digit3,
    output reg [7:0] seg,
    output reg [3:0] an
);

    reg [16:0] refresh_counter;
    wire refresh_clk;

    always @(posedge clk or posedge reset) begin
        if (reset) refresh_counter <= 0;
        else refresh_counter <= refresh_counter + 1;
    end

    // Corrected indexes: bit 1 for micro-simulation, bit 16 for physical board clock
    assign refresh_clk = (SIMULATION == 1) ? refresh_counter[1] : refresh_counter[16];

    reg [1:0] sel;
    always @(posedge refresh_clk or posedge reset) begin
        if (reset) sel <= 2'b00;
        else sel <= sel + 1;
    end

    reg [3:0] current_digit;
    always @(*) begin
        case (sel)
            2'b00: begin an = 4'b1110; current_digit = digit0; end
            2'b01: begin an = 4'b1101; current_digit = digit1; end
            2'b10: begin an = 4'b1011; current_digit = digit2; end
            2'b11: begin an = 4'b0111; current_digit = digit3; end
        endcase
    end

    // Active-High Common Cathode Map for Spartan-7 Boolean Display
    always @(*) begin
        case (current_digit)
            4'd0:    seg = 8'b00111111; 
            4'd1:    seg = 8'b00000110; 
            4'd2:    seg = 8'b01011011; 
            4'd3:    seg = 8'b01001111; 
            4'd4:    seg = 8'b01100110; 
            4'd5:    seg = 8'b01101101; 
            4'd6:    seg = 8'b01111101; 
            4'd7:    seg = 8'b00000111; 
            4'd8:    seg = 8'b01111111; 
            4'd9:    seg = 8'b01101111; 
            default: seg = 8'b00000000;
        endcase
    end
endmodule
```
## Testbench
```verilog
`timescale 1ns / 1ps

module computer_display_controller_tb;

    // 1. Declare Testbench Registers (Inputs to UUT)
    reg tb_clk;
    reg tb_reset;
    reg [7:0] tb_computer_out;

    // 2. Declare Testbench Wires (Outputs from UUT)
    wire [7:0] tb_seg;
    wire [3:0] tb_an;

    // 3. Connect to UUT using explicit, distinct names to prevent tool confusion
    computer_display_controller #(
        .SIMULATION(1)
    ) uut (
        .clk(tb_clk),                   // Connects design 'clk' to testbench 'tb_clk'
        .reset(tb_reset),               // Connects design 'reset' to testbench 'tb_reset'
        .computer_out(tb_computer_out), // Connects design input bus
        .seg(tb_seg),                   // Connects design output bus
        .an(tb_an)                      // Connects design output anodes
    );

    // 4. Clean Clock Generator (100 MHz / 10ns cycle)
    initial begin
        tb_clk = 0;
        forever #5 tb_clk = ~tb_clk;
    end

    // 5. Stimulus Vector Block
    initial begin
        // Initialize and Assert Reset immediately
        tb_reset = 1;
        tb_computer_out = 8'h00;
        #20; // Hold reset for two complete clock cycles
        
        // Release Reset
        tb_reset = 0;
        #10;

        // Test Case 1: Input Hex 2A (Decimal 042)
        tb_computer_out = 8'h2a; 
        #400;

        // Test Case 2: Input Hex 63 (Decimal 099)
        tb_computer_out = 8'h63; 
        #400;

        // Test Case 3: Input Hex FF (Decimal 255)
        tb_computer_out = 8'hFF; 
        #400;

        $display("Simulation completed successfully.");
        $stop;
    end
endmodule
```
